# Baltimore City Homicide Analysis Dashboard

This project provides an interactive Shiny dashboard for analyzing homicide trends in Baltimore City. It allows detectives, commanders, and analysts to explore data, identify patterns, and track key metrics interactively.

## Project Structure

*   **`app.R`**: The main Shiny application file, containing the UI and server logic.

*   **`Dockerfile`**: Defines the Docker environment for running the Shiny application.

*   **`run_dashboard.sh`**: A script to build the Docker image and run the Shiny application container.

*   **`data_pipeline.R` (Optional)**: A script to scrape and process raw homicide data. The `app.R` assumes this script has been run and the data is saved as `homicide_data.rds`.

*   **`homicide_data.rds` (Required)**: The processed homicide data file. This should be generated by `data_pipeline.R` or provided separately.

*   **`README.md`**: This file.
